# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
make build       # generate Tailwind CSS then compile binary → ./family-cal
make dev         # watch Tailwind + run server (requires tailwindcss CLI in PATH)
make css         # generate Tailwind CSS only
make clean       # remove binary and generated CSS
make d-build     # build docker image
make d-run       # run docker container 
go build ./...   # compile all packages without Tailwind step
```

Tailwind CLI is a standalone binary (no Node.js). Download from https://github.com/tailwindlabs/tailwindcss/releases and place in PATH.

## Architecture

Single Go binary that serves both a REST API (`/api/...`) and an embedded Alpine.js SPA (`cmd/server/web/`).

```
internal/
  db/           SQLite connection (WAL mode) + embed-based migration runner
  auth/         JWT httpOnly cookie auth, bcrypt, login/logout/setup handlers, middleware
  user/         Family member CRUD, color, notification preferences
  calendar/     Event CRUD + recurrence expansion + event exceptions
  notification/ Hourly goroutine: emails upcoming events via SMTP, deduplication log
  api/          chi router wiring all handlers + YAML/env config loader
cmd/server/
  main.go       Entry point: opens DB, wires services, starts scheduler, serves SPA
  web/          Embedded frontend (index.html + Alpine.js + Tailwind output.css)
```

**DB migrations** live in `internal/db/migrations/`. The runner applies them in filename order on startup; each migration is tracked in `schema_migrations`.

**Recurrence**: recurring events store a `recurrence_id` pointing to a `recurrence_rules` row. `GET /api/events?from=&to=` expands them in-memory. Edits/deletes use `scope=this|future|all` — `this` writes to `event_exceptions`, `future` truncates the rule's `until` date, `all` modifies the base event directly. Recurring event occurrence IDs are `{baseID}-{YYYY-MM-DD}`.

**First-run setup**: on startup with an empty `users` table, the server prints a one-time token to stdout. `POST /api/auth/setup` accepts that token to create the first admin account.

**Frontend**: single `index.html` with Alpine.js managing all state. Hash-based routing via Alpine `x-show`. No build step for JS — Alpine is vendored at `web/static/alpine.min.js`. Tailwind `output.css` is gitignored and generated by `make css`.
